1 . What is DOM and what is the purpose of DOM?

Answer :

DOM stands for Document Object Model.

All HTML webpages are referred to as documents. The Document Object Model represents each of these web pages in a tree like structure to make it easier to access and manage the elements.Because of it , programs can change the document structure, style, and content. The DOM represents the document as nodes and objects, that way, programming languages can interact with the page.

The Document Object Model is significant in speeding up web page loads.

2 . How will you select HTML elements using DOM? Name the DOM methods.

Answer :

There are 5 ways to select elements in a DOM using selectors.They are given below :

They are given in below :

1 . querySelector() - It returns the first element within the document that matches the specified selector, or group of selectors. If no matches are found, null is returned.

2 . querySelectorAll() - It returns a static NodeList representing a list of the document's elements that match the specified group of selectors.

3 . getElementById() - It returns an element object representing the element whose id property matches the specified string.

4 . getElementsByClassName() - This method returns a collection of elements with a specified class name(s),returns an HTMLCollection and property is read-only.

5 . getElementsByTagName() - This method returns a collection of all elements with a specified tag name,returns an HTMLCollection and property is read-only.

3 . What is event bubble?

Answer :

When an element like a button is clicked, an event is directed to the element, if an event handler is set for the element, the event handler is triggered.

In bubbling, when an event happens, the handler of the innermost element runs, then the event bubbles up to the elements parent. If an event handler is set for the parent, this event handler is triggered and then the further ancestor elements. In other words, events bubble up or propagate the DOM tree upwards.

4 . What is a callback function and why will you use it?

Answer :

Any function that is passed as an argument to another function so that it can be executed in that other function is called as a callback function.

JavaScript runs code sequentially in top-down order. However, there are some cases that code runs after something else happens and also not sequentially. Callback function make sure that a function is not going to run before a task is completed but will run right after the task has completed. It keeps us safe from problems and errors.
